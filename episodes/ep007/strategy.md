# Strategy

### Hook angle

DHH's Codex take is not a conversion story; it is a workflow audit where the typing moved to the agent and the responsibility stayed with the human.

### Why this works

The funny part is procedural. A famously hands-on programmer does not become a futurist mascot; he becomes the reviewer of a very fast junior contractor with shell access. The Mike Judge / Gary Larson version is a software team treating "agent-first development" like a new procurement process.

### Format recommendation

**Mockup + kinetic**. Do not depend on stock b-roll. Use fake code, JIRA, Slack, and calendar artifacts so the episode has actual visual texture even offline.

### Retention structure

1. **Hook (0-5 s):** DHH used to type the code himself; now the agent starts the work order.
2. **Body:** Contrast autocomplete with agents, then show the workflow as tickets, diffs, tests, and review.
3. **Callback / button:** Codex can draft the patch; the human still owns taste and the merge.

### Scene count and target duration

- Recommended scene count: 10 scenes
- Target duration: 35-42 seconds
- Rationale: Fast enough for a timely tech take, but with enough visual variety to make the process legible.

### B-roll search terms

Not applicable. This episode should not use b-roll unless a strong local clip is already available.

### Brand voice notes for script agent

Keep the DHH angle factual and dry. Do not dunk on him and do not treat AI coding as magic. The joke is the office procedure around the agent: tickets, approvals, review, and the merge button.
