# Claim Review

## Verdict

Approved for draft review. The episode makes one factual claim: DHH moved from not using AI tools to write code toward an agent-first workflow. That claim is supported by current public sources.

## Sources Checked

- The Pragmatic Engineer, "DHH's new way of writing code," published April 8, 2026: states that six months earlier DHH said he typed his own code, and that the episode discusses why he now takes an agent-first approach and barely writes code by hand. It also summarizes his workflow as two models running with Neovim in the center.
- DHH, "Promoting AI agents," published January 7, 2026: DHH wrote that AI agents had "really came alive" for him when models improved and agents gained tools such as terminal control, tests, web search, and plain-English skills.
- OpenAI, "Introducing Codex," published May 16, 2025: describes Codex as an AI coding agent for offloading well-scoped tasks such as refactoring, tests, scaffolding, bug fixes, and documentation.

## Claims

### DHH used to type the code himself.

Supported. The Pragmatic Engineer episode summary says that six months earlier, on Lex Fridman, DHH described not using AI tools to write code and typing out all his code.

### Agents changed when they got terminal, tests, and diff-oriented workflows.

Supported with wording adjustment. DHH's January 2026 post says agents became materially useful when models improved and tools expanded beyond pure reasoning, including terminal control and test-running. The "diff" language is an episode framing of review workflow, not a direct quote.

### Codex now starts the draft before the programmer opens the editor.

Supported as commentary, not literal attribution. The Pragmatic Engineer summary frames DHH's current approach as agent-first. The script should not imply DHH uses only OpenAI Codex; it uses Codex as the episode's agentic-coding shorthand.
